# ShitCode Fest 2025

Добро пожаловать в репозиторий «абсурдного программирования». Здесь лежат решения задач, реализованные через максимально неэффективные, странные и бюрократизированные архитектуры.

## Общие принципы
Все решения следуют архитектуре **Client-Server**, даже если задача решается одной строчкой. Мы используем HTTP-запросы для элементарных операций и алгоритмы типа **Bogo-Sort** для создания искусственной нагрузки на CPU и мозг.

## Задание 1: Максимально неэффективная сортировка (`1.py`)

### Суть
Программа сортирует список чисел, но делает это так, чтобы процессор молил о пощаде.

### Как это работает
1. **Клиент**: Запускает встроенный HTTP-сервер в отдельном потоке.
2. **Алгоритм**: Используется вариация *Selection Sort*. Для поиска каждого следующего минимального (или максимального) элемента клиент обращается к серверу.
3. **Серверный Bogo-Selection**: 
    - Сервер пытается угадать нужный элемент.
    - Чтобы подтвердить, что кандидат правильный, сервер запускает **Paranoia Mode**: он перемешивает весь массив до тех пор, пока он не станет отсортированным, 2000 раз подряд!
    - Внутри каждой итерации считается бесполезное число Фибоначчи для прогрева атмосферы.

### Запуск
```bash
python3 1.py
```
Программа попросит выбрать тип сортировки (1 или 2) и выведет результат после того, как "сожжет" достаточно циклов процессора.

## Задание 2: Робин Гуд и бюрократический аудит (`2.py`)

### Суть
Решение задачи «Robin Helps». Робин забирает золото у богатых ($a_i \ge k$) и отдает по 1 единице беднякам ($a_i = 0$), пока золото не кончится.

### Как это работает
1. **Бюрократический сервер**: Каждое сравнение чисел проходит через полноценный HTTP-аудит на случайном порту.
2. **Эндпоинты**:
    - `/rich`: Проверяет условие "богатства".
    - `/poor`: Проверяет, является ли человек бедняком.
3. **Bogo-Max Audit**:
    - Чтобы проверить $a \ge b$, сервер кладет их в список и делает `random.shuffle`, пока $a$ не окажется на первом месте в отсортированном по убыванию списке.
    - Для "стабильности" аудит повторяется **50 раз** для каждого человека.

### Запуск
Программа ожидает данные в формате соревновательного программирования через стандартный ввод (`stdin`).

**Способ А (через файл):**
```bash
python3 2.py < sample_input.txt
```

**Способ Б (вручную):**
1. Запустите `python3 2.py`.
2. Вставьте входные данные (например):
   ```
   1
   5 2
   3 1 0 2 5
   ```
3. Нажмите **Enter**, а затем **`Ctrl+D`** (сигнал конца ввода для macOS/Linux).
